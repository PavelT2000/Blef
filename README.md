# Блеф — многопользовательская веб-игра

Онлайн-версия карточной игры **«Блеф»** (Liar's Poker / Perudo-подобные заявки по покерным комбинациям). Несколько независимых комнат, вход по коду или ссылке, синхронизация через **Server-Sent Events (SSE)**, таймер хода **30 секунд** с автоматическим действием на сервере. Неактивные комнаты удаляются через **5 минут**.

## Быстрый старт

```bash
pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

Откройте в браузере: [http://127.0.0.1:8000](http://127.0.0.1:8000)

На Linux (Orange Pi, VPS) для чистого venv лучше ставить только зависимости игры:

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements-app.txt
uvicorn main:app --host 0.0.0.0 --port 9000
```

Для production без `--reload`:

```bash
uvicorn main:app --host 0.0.0.0 --port 9000 --workers 1
```

Убедитесь, что на сервере есть каталоги `templates/` и `static/css/`.

## Функциональность

### Комнаты

| Возможность | Описание |
|-------------|----------|
| **Несколько комнат** | Каждая игра изолирована; комнаты живут в памяти сервера |
| **Создание комнаты** | Кнопка на главной → 6-символьный код (латиница + цифры) |
| **Вход по коду** | Ввод кода и имени на главной |
| **Вход по ссылке** | `/join/{КОД}` — код подставляется автоматически |
| **Приглашение** | На странице игры ссылка «пригласить» ведёт на `/join/{КОД}` |
| **Лимит игроков** | 2–13 человек в комнате |
| **Пустая комната** | Когда все игроки нажали «Покинуть комнату», комната **удаляется** с сервера |
| **Неактивная комната** | Нет входов, стартов и ходов **5 минут** — комната удаляется (игрокам уходит `room_closed` по SSE) |

### Игровой процесс

1. **Лобби** — игроки собираются; лимит карт для выбывания **подбирается автоматически** (см. ниже). Любой игрок может запустить партию.
2. **Раунд** — у каждого по картам на руках; по очереди заявляют покерную комбинацию сильнее предыдущей или говорят **«Не верю»**.
3. **Вскрытие** — проверка, есть ли заявленная комбинация среди **всех** карт за столом; проигравший получает штрафную карту из колоды.
4. **Выбывание** — набрал заданное число карт → выбыл.
5. **Победа** — остался один активный игрок.

### Таймер хода (30 сек)

Считается **на сервере**; клиент показывает обратный отсчёт по полю `turn_deadline`.

| Ситуация | Автодействие |
|----------|----------------|
| На столе **есть** ставка | **«Не верю»** (CHALLENGE) |
| Ставок **ещё нет** | Минимальная заявка: **Старшая карта 2** |

### Таймер вскрытия (10 сек)

После «Не верю» — пауза **10 секунд**, затем сервер сам завершает вскрытие (раздача штрафа, новый раунд). Кнопка «Пропустить» по-прежнему доступна.

### Выход и сброс

- **Покинуть комнату** — игрок удаляется; если активных осталось &lt; 2 во время игры — игра завершается с победителем.
- **Новая игра (лобби)** — сброс партии в лобби **с теми же игроками** в той же комнате.

## Правила комбинаций

Иерархия (от слабой к сильной): Старшая карта → Пара → Две пары → Сет → Стрит → Фулл-хаус → Флеш → Каре → Стрит-флеш → Флеш-рояль.

Сравнение ставок — `bid_comparison.py` (по `bets.txt`). Проверка на столе — `combination_check.py` (все карты активных игроков).

### Параметры заявки

| Комбинация | Поля | Как читать |
|------------|------|------------|
| Старшая карта, пара, сет, каре | `rank_primary` | **Точное** достоинство (нужно столько карт этого ранга на столе, сколько требует комбинация). |
| Две пары, фулл-хаус | `rank_primary`, `rank_secondary` | **Точные** достоинства (для фулла: тройка = primary, пара = secondary). |
| Стрит, стрит-флеш | `rank_primary` | **Нижняя граница включительно (≥)** — младшая карта найденного стрита не ниже выбранного ранга; карты ниже границы не учитываются. Учитывается и «колесо» 2–3–4–5–Т, если граница ≤ 2. |
| Флеш | `rank_primary`, опц. `suit` | **Нижняя граница включительно (≥)** — пять карт на столе с рангом **не ниже** выбранного; при указании масти все пять одной масти. |
| Флеш-рояль | опц. `suit` | Фиксированные 10–В–Д–К–Т; масть в заявке опциональна и усиливает ставку при сравнении. |

**Ответ на вопрос про границу:** для стрита, флеша и стрит-флеша нижняя граница **включительная** — в коде проверка `start >= rank_primary` / `rank >= rank_primary`. Для пары, сета и т.п. поле — не граница, а **точный** номинал.

На форме хода подсказка меняется при выборе типа комбинации.

## API

### Комнаты

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/rooms/create` | Создать комнату → `{ room_code, join_url }` |
| `GET` | `/api/rooms/{room_code}` | Статус комнаты (лобби/игра, число игроков) |

### Игра

| Метод | Путь | Тело (JSON) |
|-------|------|-------------|
| `POST` | `/api/game/join` | `{ name, room_code }` |
| `POST` | `/api/game/leave` | `{ player_id, room_code }` |
| `POST` | `/api/game/start` | `{ player_id, room_code }` |
| `POST` | `/api/game/action` | `{ player_id, room_code, action, ... }` |
| `POST` | `/api/game/finish-showdown` | `{ player_id, room_code }` |
| `POST` | `/api/game/reset` | `{ player_id, room_code }` |
| `GET` | `/api/game/stream/{room_code}/{player_id}` | SSE-поток состояния |

### Страницы

| URL | Назначение |
|-----|------------|
| `/` | Главная: создать / войти |
| `/join/{room_code}` | Вход с подставленным кодом |
| `/game/{room_code}/{player_id}` | Игровой стол |

### Модули

| Файл | Роль |
|------|------|
| `main.py` | Маршруты FastAPI, HTML-страницы, делегирование в `room_manager` |
| `room_manager.py` | Словарь комнат `{код → GameState}`, asyncio-таймеры хода и вскрытия, broadcast SSE, удаление пустых комнат |
| `game_logic.py` | `handle_player_action`, `finalize_showdown`, `next_player`, маскировка карт в `get_masked_game_state` |
| `schemas.py` | Pydantic-модели: `Card`, `Bid`, `Player`, `GameState` |
| `requests.py` | DTO входящих запросов |
| `enums.py` | `CombinationType`, `CardSuit`, `GameStatus` |
| `deck.py` | Колода 52 карты, раздача, перетасовка рук между раундами |
| `sse_manager.py` | Подписки игроков: `{ player_id → set(Queue) }` |
| `config.py` | Таймеры, лимиты игроков, `compute_elimination_limit()`, `ROOM_INACTIVITY_SECONDS` |

### Состояние игры (`GameState`)

- `status`: `LOBBY` → `PLAYING` → `SHOWDOWN` → снова `PLAYING` или `GAME_OVER`
- `players`, `current_player_idx`, `bid_history`, `deck`
- `turn_deadline` / `turn_generation` — серверный таймер хода
- `showdown_deadline` — автозавершение вскрытия
- Поля вскрытия: `showdown_loser_id`, `showdown_message`, `showdown_combination_found`

Клиенту в SSE уходит **замаскированное** состояние: чужие карты как `{is_unknown: true}`, размер колоды как `deck_count`, без внутреннего `turn_generation`.

### Поток хода

1. Игрок отправляет `POST /api/game/action`.
2. `room_manager` отменяет таймер, вызывает `handle_player_action`.
3. При `RAISE` — `next_player`, планируется новый таймер.
4. При `CHALLENGE` — `SHOWDOWN`, таймер вскрытия 10 с.
5. По таймауту хода — `build_auto_turn_action` → тот же pipeline.
6. `broadcast_room` шлёт JSON всем подписчикам комнаты.

### Масштабирование и ограничения

- Комнаты хранятся **только в RAM**; перезапуск сервера обнуляет все игры.
- Один процесс uvicorn — достаточно для учебного/локального использования.
- Для production нужны Redis/БД для комнат и отдельный воркер таймеров.

### Лимит выбывания (авто)

На сервере (`config.compute_elimination_limit`):

1. `raw = trunc(52 / число_игроков)` → `52 // num_players`
2. `limit = clamp(raw, 4, 12)` — не меньше **4** и не больше **12** карт

| Игроков | raw | Лимит |
|---------|-----|-------|
| 2 | 26 | **12** |
| 4 | 13 | **12** |
| 6 | 8 | **8** |
| 10 | 5 | **5** |
| 13 | 4 | **4** |

В лобби в SSE приходит `planned_elimination_limit` до старта партии.
